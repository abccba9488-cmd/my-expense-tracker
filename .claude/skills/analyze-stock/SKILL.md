# 個股專業分析

針對單一股票代號，整合本站資料庫的客觀數字 + 即時網路搜尋的市場資訊，輸出一份基金經理人風格的估值分析報告。**數字一律用本站資料算，不要憑空猜**；網路搜尋只用來補資料庫沒有的東西（同業本益比、外資EPS預估、新聞、籌碼）。

## 0. 決定 base URL

本站 2026-07-22 起以本機為主，雲端（Zeabur）僅保留為備援、不保證資料是最新的。**先試 `GET http://localhost:5000/api/stats`**（timeout 設短一點，例如 3 秒）：
- 連得到 → 這次分析全程用 `http://localhost:5000` 當 base URL
- 連不到（逾時、拒絕連線，或不在能存取這台機器 localhost 的環境）→ 改用 `https://stock-market.zeabur.app` 當 base URL，並在報告開頭註明「資料來源：雲端備援站台，可能非最新」

## 1. 解析股票代號

使用者若只給公司名稱沒給代號，呼叫 `GET <base URL>/api/market/summary`（回傳全部約1700支股票快照），用 `name` 比對找出 `code`。若使用者已給 4 位數代號可直接跳過這步。

## 2. 抓本站資料（依序呼叫，base URL 為上一步決定的那個）

- `GET /api/stocks/<code>` — 名稱、市場（TWSE/TPEX）、產業
- `GET /api/stocks/<code>/prices?days=250` — 近一年日線（含 OHLCV），用來看週線/日線趨勢
- `GET /api/stocks/<code>/revenue` — 全部月營收歷史（含 `revenue_yoy`/`revenue_mom`）
- `GET /api/stocks/<code>/financials` — 全部季財報歷史（含 `eps`/`operating_income`/`net_income`）

從這四份資料自己算出：
- **目前股價**：`prices` 最後一筆的 `close`，附上日期
- **本益比**：取最新一筆 `financials`。若該季是 Q1–Q3，`PE = close / (eps / quarter × 4)`（年化）；若是 Q4，`PE = close / 該年四季EPS加總`
- **營收年增趨勢**：`revenue` 最近 3-6 個月的 `revenue_yoy` 變化方向（加速/減速/由負轉正）
- **營收預估股價**（本站特有指標，反映「營收動能是否已反映在股價上」）：`(最新月營收 / 最新季營收) × 最新季EPS × 240`，跟目前股價比較算出倍數（本站定義 ≥1.5倍 為飆股訊號，需同時營收年增 ≥20%）
- **轉折訊號**：EPS/營收是否剛由衰退轉成長、或由虧轉盈（比對最近兩期）

若以上端點任何一個回傳空陣列（新股、剛上市、資料還沒補齊），在報告中明確標註「本站資料庫缺X」，不要用網路搜尋的數字偷偷頂替本站本來該有的客觀數字。

**資料合理性檢查**：算出上述數字後，先過一輪 sanity check，抓出數字本身可能有問題（爬蟲/單位/計算異常），不要照樣端出去：
- 毛利率 >100% 或 <-50%
- 流動比率 <0
- 負債比率 >100%（非金融業）
- ROE >100%
- 相鄰年度/季度淨利率波動 >30 個百分點（可能是一次性損益，不是本業變化）

任何一項觸發時，在報告對應數字旁加註「⚠ 數字異常，建議人工核對」，不要略過不提或自行「合理化」修改數字。

## 3. 即時網路搜尋（WebSearch / WebFetch）

只搜尋本站資料庫沒有、且分析必需的資訊：
1. 同業競爭者目前的「預估本益比」（至少 2-3 家可比較公司）
2. 外資（Morgan Stanley、大摩、外資券商等）對這檔股票今年全年 EPS 的預估值
3. 近 1-3 個月是否有重大新聞影響股價（新訂單、產能、價格、地緣政治、客戶集中度等）
4. 近 3 個月法人（外資/投信/自營商）籌碼是否在累積或出脫
5. 是否有相同或上下游產業、還沒漲但題材相關的「補漲」候選股

若搜尋不到，在報告中明確寫「未搜尋到相關資訊，以下評估僅依本站財務數據」，不要編造。

## 4. 定價計算

以步驟2算出的本益比與步驟3找到的同業本益比/外資EPS預估為基礎，設 **20% 折價** 作為安全邊際：
- **便宜價**：`min(本站EPS估值, 外資EPS估值) × 同業平均PE × (1-20%)`
- **合理價**：`本站/外資EPS共識 × 同業平均PE`
- **昂貴價**：`樂觀EPS情境 × 同業PE上緣`

標註目前股價落在哪個區間（便宜/合理/昂貴/低於便宜價）。EPS為負或本益比算不出來時，改用營收成長性與轉機題材定性描述，不要硬算出無意義的本益比數字。

## 5. 技術面（中長期，週線搭配日線）

用步驟2抓到的日線資料，描述：近一年高低點區間、目前位置（相對高/低點百分位）、近期趨勢（站上/跌破關鍵均線可用 20/60 日均估算）、量價是否同步。使用者投資以中長期為主，優先看週線級別的趨勢方向，日線只用來確認近期動能。

## 6. 輸出格式

用繁體中文，結構固定如下：

```
【公司】代號 名稱（市場/產業）
【目前股價】XX元（資料日期）｜【本站本益比】XX倍｜【分析執行時間】YYYY-MM-DD

📊 財務數據（本站資料庫）
- 最新季EPS：X元（YoY X%）｜最新月營收：X仟元（YoY X%）
- 營收預估股價：X元（現價的 X 倍）

🌐 市場資訊（即時搜尋）
- 同業本益比：A公司X倍／B公司X倍
- 外資全年EPS預估：X元
- 近期新聞／籌碼：...

💰 定價區間
便宜價 X元 ｜ 合理價 X元 ｜ 昂貴價 X元 ｜ 目前股價落在：[區間]

📈 技術面
[週線趨勢] + [日線動能]

🔍 結論與風險
是否適合投資、目標價、主要風險、相關補漲候選股
```

缺資料的欄位寫「本站缺此資料」或「未搜尋到」，不要留白也不要編數字。

**敘述性文字一律要帶具體數字**：報告裡任何描述趨勢/動能的句子都不能只寫方向詞，例如不要寫「營收持續成長」「動能轉弱」，要寫「營收YoY +18.1%，OO訂單驅動」「6月YoY轉為-9.0%，較前月+2.1%明顯轉弱」這種帶數字+原因（若查得到原因）的句子。有明確消息來源佐證的敘述，附上日期與來源（例如「工商時報，2026-07-04」），讓使用者可以自行回頭核對。
